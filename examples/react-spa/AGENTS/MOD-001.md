# MOD-001 — API Layer

## Scope

```
src/services/api.ts         ← typed wrappers around every backend endpoint
src/services/auth.ts        ← login/logout/refresh token calls
src/utils/http.ts           ← base fetch wrapper (attaches tokens, handles 401)
src/types/api.ts            ← shared request/response interfaces
```

## What This Module Does

All communication with the backend goes through this module. It provides typed async functions that wrap `fetch` calls, attach auth tokens, handle 401 auto-refresh, and normalize error shapes before they reach the UI.

## Key Files — What to Know Before Editing

### `src/utils/http.ts`
- All API calls must go through `http.get/post/put/delete` — never call `fetch()` directly
- Automatically attaches the Bearer token from the auth store
- On a 401 response, calls `auth.refresh()` once and retries. If the retry also fails, it dispatches a `SESSION_EXPIRED` event and rejects
- **Do not add business logic here** — this is infrastructure only

### `src/services/api.ts`
- Each function maps 1:1 to a backend endpoint
- Return types are defined in `src/types/api.ts` — add types there, not inline
- Error responses are always `{ error: string; code: string }` — the wrapper normalizes them

### `src/types/api.ts`
- The shared contract. Every other module imports types from here
- **Breaking changes here break every module** — coordinate with team before changing existing interfaces

## Critical Constraints

- Never store tokens in `localStorage` — they live in `httpOnly` cookies (managed by the server) and the in-memory auth store
- Never `await fetch()` outside this module
- All error paths must throw a typed `ApiError` — never swallow errors silently
