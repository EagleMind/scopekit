## Contract

- `http.get/post/put/delete` is the only sanctioned way to call the backend. @anchor src/utils/http.ts
- Every API function maps 1:1 to an endpoint and returns a type from `api.ts`. @anchor src/services/api.ts
- Failed calls throw a typed `ApiError` — `{ error, code }`. @anchor src/types/api.ts::ApiError

## Invariants

- Never call `fetch()` directly outside this module — always go through `http`. @anchor src/utils/http.ts
- On a 401, `http` calls `auth.refresh()` once and retries; a second failure dispatches `SESSION_EXPIRED`. @anchor src/utils/http.ts
- Tokens live in httpOnly cookies + the in-memory auth store — never `localStorage`. @anchor src/services/auth.ts
- Shared types in `api.ts` are imported everywhere; coordinate before changing them. @anchor src/types/api.ts

## Internal

- `http.ts` is infrastructure only — keep business logic out of it. @anchor src/utils/http.ts
- Error responses are normalized to `{ error, code }` by the wrapper before they reach callers. @anchor src/services/api.ts
